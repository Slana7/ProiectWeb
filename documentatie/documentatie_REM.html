Documentație Tehnică – REM (Real Estate Management)

Scopul Aplicației

REM este o aplicație web care facilitează gestionarea anunțurilor imobiliare. Utilizatorii pot posta proprietăți, le pot salva ca favorite și pot comunica direct prin sistemul de mesaje. Harta integrată permite filtrarea după locație și facilități.

Arhitectura Generală

Aplicația urmează modelul MVC:
  • Model: Clase PHP care interacționează cu baza de date (User, Property, Message etc.)
  • View: Fișiere PHP din folderul views/ care generează interfața HTML
  • Controller: Fișiere din src/Controllers care preiau cererile utilizatorilor

Baza de Date

Aplicația folosește o bază de date relațională MySQL/PostgreSQL. Tabele esențiale:
  • users – conturi utilizatori
  • properties – anunțuri imobiliare
  • favorites – relație many-to-many între users și properties
  • messages – conversații între utilizatori

Fluxuri Principale

  1. Autentificare: Utilizatorul completează email/parolă, datele sunt validate de AuthController
  2. Adăugare anunț: Datele sunt transmise către PropertyController și salvate în baza de date
  3. Mesaje: MessageController preia și salvează conversațiile între utilizatori

Tehnologii Folosite

  • Backend: PHP (fără framework)
  • Frontend: HTML, CSS, JavaScript
  • Geolocalizare: Leaflet.js pentru hărți interactive
  • Bază de date: PostgreSQL sau MySQL

Îmbunătățiri Viitoare

  • Refactorizare în Laravel/Symfony pentru o arhitectură scalabilă
  • API REST pentru integrare cu aplicații mobile
  • Notificări în timp real (ex: WebSockets pentru mesaje noi)
  • Testare automată (unit și integrare)
